Funktionalitäten:


- Charaktererstellung: Aussehen der Kuh, verschiedene Fellfarben, Hörner etc. (wird zunächst nicht implementiert, es gibt erst mal einen generischen Charakter)



Szene darstellen
----------------

Dazu gehört:
- Auslesen der benötigten Werte aus der DB. Dies ermöglich flexibel beliebige Szenen/Level in der DB zu speichern.

- Hintergrundgrafik darstellen
	- Anhand der DB Grafik bestimmen und mit dieser das Canvas füllen

- Interaktive Objekte darstellen Objekt 

- Kuh an der Startposition darstellen

- Bei Klick auf eine leere Position in der Scene wird die Kuh bewegt, "läuft zur Position" (Laufanimation), für ersten Prototypen statisches Bild.

- Bei Klick auf ein Objekt wird das Event ausgeführt, welches in der DB mit dem Objekt verknüpft ist , z.B. Animation, Sound, spawnen neuer Objekte, selbst löschen, Wechsel zu Szene XY. etc. Die konkrete Umsetzung der Events ist noch nicht vollständig definiert.

- Bei Szenenwechsel wird die Szene gelöscht und eine neue wie oben generiert. So können mehrere Szenen zu einem Gesamtspiel verknüpft werden.

- Sounds bei Bedarf abspielen, welche das sind wird aus der DB ausgelesen.

- Registrierte User: 
	Registrierung
		- Prüfen der eingegebenen Daten wie Name, E-Mail, Passwort auf Plausibilität und Sicherung gegen SQL Injections.
		- Erstellen eines neuen Benutzerkontos
			- Speichern der Daten in einem Datensatz.
				- Sicheres Speichern des Passwortes (Stichwort: hash)
	Login
		- Prüfen der Login Daten auf Korrektheit. Plausi- und Sicherheitsprüfungen wie bei der Registrierung.
		- Einloggen des Users

	Speichern des Spielstandes
		- Speichern der aktuellen Szenen ID in der DB
	Laden des Spielstandes
		- Auslesen der aktuellen Szenen ID in der DB.
	
	Hinweis zum Laden und Speichern: Im ersten Schritt wird nur festgehalten in welcher Szene sich der Spieler befindet


- Installationsscript: Erstellt alle notwendigen MySQL Tabellen auf Knopfdruck, für eine saubere und einfache Installation.
